# SnakeArena

基于 **Pygame** 的网格贪吃蛇环境，集成多种 **深度强化学习** 与 **经典规划 / 启发式** 控制策略，并提供统一启动器、人机对战、多 AI 混战，以及可选的 **Web（ONNX）** 复刻版本。

---

## 项目概览


| 模块                          | 说明                                       |
| --------------------------- | ---------------------------------------- |
| `game_env.py`               | 核心环境：`SnakeGameAI`（网格、随机墙、食物、碰撞、奖励与超时终止） |
| `agent_*.py` + `model_*.py` | 各算法的训练循环与网络定义（PyTorch）                   |
| `play_*.py`                 | 单脚本演示：人类操作、规划算法自动玩、或加载模型推理               |
| `ui_*.py`                   | 图形界面：主菜单、单人、双人、对战 AI、多蛇混战、算法适配层          |
| `snake-web/`                | 浏览器端复刻；可通过脚本将 `.pth` 导出为 ONNX            |


**动作空间**：每步在「直走、右转、左转」三选一，编码为 `[1,0,0]` / `[0,1,0]` / `[0,0,1]`，与环境中顺时针方向序 `right → down → left → up` 一致。

**观测（强化学习）**：统一的 **11 维向量**（前方/左/右危险、朝向 one-hot、食物相对方位）；逐项含义见下文 **「11 维状态向量」** 小节中的表格。

## 架构说明

### 1. 环境层（`game_env.py`）

- **网格**：`ROW=30`，`COL=60`，窗口与单元格由常量推导。
- **随机障碍**：约占格子总数 1% 的墙。
- **奖励**：吃到食物 `+10`，死亡 `-10`，其余步默认 `0`。
- **终止**：撞墙/撞身/越界，或步数超过 `100 * len(snakes) + 100`（防止训练时无限绕圈）。
- `**is_training`**：`True` 时高帧率便于训练；`False` 时按得分阶梯调速，便于人眼观察。

### 2. 强化学习训练（`agent_*.py`）

各脚本自包含训练主循环：与环境交互、写日志到 `log/log_<算法>/`，模型保存到 `model/model_<算法>/`，支持断点续训与周期性保存；**PPO / TRPO / A2C** 在环境中额外叠加了「曼哈顿距离靠近食物」的 shaping 奖励（见对应 `agent_*.py`）。

### 3. 规划与启发式（`play_*.py`）

在**当前蛇头、蛇身、墙、食物**上建图或采样，输出下一步动作；多数脚本在无路可走时使用「忽略蛇尾」的碰撞检测做安全兜底。

### 4. UI 与对战（`ui_launcher.py` 等）

- `**ui_launcher.py`**：主菜单 → Single Player / Two Players / Play vs AI / AI Brawl。
- `**ui_pvp_core.py**`：双人规则与渲染（含与 AI 控制器对接的步进逻辑）。
- `**ui_ai_controllers.py**`：将各算法封装为统一 `next_action(game_like)`；对战时通过 `_ProxyGame` 把对手占用格并入「墙」，避免规划器误把对手当可缩进的蛇尾。
- `**ui_vs_ai.py**`：从网格中选择算法后进入人机局。
- `**ui_ai_brawl.py**`：2～4 条蛇，每条蛇独立选算法后混战。
- `**ui_high_score.py**`：单人模式最高分持久化到 `high_score.txt`。

### 5. Web 子项目（`snake-web/`）

静态页面 + JS 复刻玩法；RL 侧依赖导出的 ONNX。详见 `[snake-web/README.md](snake-web/README.md)`。

## 算法详解

### 与本项目相关的 MDP 约定


| 项目    | 说明                                                                         |
| ----- | -------------------------------------------------------------------------- |
| 状态    | 强化学习使用 **11 维手工特征**（非像素）；各维含义见「11 维状态向量」表                                  |
| 动作    | `straight / right / left` → 环境内前进或 90° 转弯                                  |
| 环境奖励  | 吃食物 **+10**，死亡 **-10**，普通步 **0**；另有步数超时终止                                  |
| 额外奖励  | **仅 PPO / TRPO / A2C**：未结束时，曼哈顿距离靠近食物 **+0.5**，远离 **-0.2**                 |
| UI 推理 | `RLPolicyController` 加载 `model/model_<算法>/`（优先 `best_*.pth`）；失败则回退 **BFS** |


### 11 维状态向量（与 `get_state` 顺序一致）


| 维    | 语义         | 计算要点                           |
| ---- | ---------- | ------------------------------ |
| 0    | 正前方危险      | 当前朝向前一格 `is_collision`         |
| 1    | 右转后前方危险    | 相对坐标映射后检测                      |
| 2    | 左转后前方危险    | 相对坐标映射后检测                      |
| 3–6  | 朝向 one-hot | `left / right / up / down` 四选一 |
| 7–10 | 食物方位       | 左 / 右 / 上 / 下 四个布尔             |


未编码整条蛇身与全图墙分布，属常见教程式**部分可观测**特征。

### 算法总览（13 种，便于横向对比）


| 算法              | 大类     | 训练/演示入口               | 需预训练 | 每步决策方式                             | 最优路径保证                 | 典型复杂度（本网格）                           |
| --------------- | ------ | --------------------- | ---- | ---------------------------------- | ---------------------- | ------------------------------------ |
| **DQN**         | 值函数 RL | `agent_DQN.py`        | 是    | 对 Q 网络输出取 `argmax`，映射为直走/右转/左转     | 无（策略由训练得到）             | 训练耗时；推理每步 O(1)                       |
| **DDQN**        | 值函数 RL | `agent_DDQN.py`       | 是    | 对在线 Q 网络取 `argmax`，映射为三动作          | 无                      | 训练耗时；推理 O(1)；目标网络每 10 局硬同步           |
| **Dueling DQN** | 值函数 RL | `agent_DuelingDQN.py` | 是    | 对 Dueling 头合成的 Q 取 `argmax`，映射为三动作 | 无                      | 训练耗时；推理 O(1)；网络为 V+A 双支路             |
| **PPO**         | 策略 RL  | `agent_PPO.py`        | 是    | 训练时按 Softmax 采样；UI 对策略概率取 `argmax` | 无                      | 每局结束更新；`ppo_epochs=10`               |
| **TRPO**        | 策略 RL  | `agent_TRPO.py`       | 是    | 训练时按 Softmax 采样；UI 对策略概率取 `argmax` | 无                      | 每局结束更新；KL 约束 + 共轭梯度 + 线搜索            |
| **A2C**         | 策略 RL  | `agent_A2C.py`        | 是    | 训练时按 Softmax 采样；UI 对策略概率取 `argmax` | 无                      | 每局结束更新；Actor 1 epoch、Critic 10 epoch |
| **BFS**         | 图搜索    | `play_bfs.py`         | 否    | 队列求最短路，执行路径首格对应的三动作                | 边权为 1 时步数最少            | 每步约 O(ROW×COL)                       |
| **Dijkstra**    | 图搜索    | `play_Dijkstra.py`    | 否    | 堆求最短路，执行路径首格对应的三动作                 | 边权均为 1 时，路径步数与广度优先搜索相同 | 每步约 O(ROW×COL)；便于改为加权边               |
| **A**           | 图搜索    | `play_Ax.py`          | 否    | `f=g+h`（h 为曼哈顿距离），执行路径首格对应的三动作     | 启发可采纳时路径最短             | 每步约 O(ROW×COL)；启发式常减少扩展节点数           |
| **RRT**         | 采样规划   | `play_RRT.py`         | 否    | 随机扩展树，沿树走向食物的第一步转为三动作              | 无                      | 每步最多 `max_iter=1500` 次扩展             |
| **RRT**         | 采样规划   | `play_RRTx.py`        | 否    | 带重连优化的树，执行优化后路径首格对应的三动作            | 无；迭代足够时路径趋向更短          | 每步最多 1500 次扩展；含重连，计算量通常大于基础 RRT      |
| **APF**         | 反应式    | `play_APF.py`         | 否    | 在前/右/左三候选格中选总势能最低且可行走者             | 无                      | 每步 O(1)                              |
| **DWA**         | 反应式    | `play_DWA.py`         | 否    | 对直走/右转/左转三动作加权打分，取最高分              | 无                      | 每步 O(1)                              |


> **命名**：界面 **Ax** 对应 `play_Ax.py`（**A**）；界面 **RRTx** 对应 `play_RRTx.py`（**RRT**，非动态障碍场景的 RRT-X 论文算法）。

### A. 值函数强化学习（DQN 族）

**共性**：网络 `11 → 256 → ReLU → 3`；TD 目标形如 `y = r + γ·max_a' Q(s',a')`（终止步 `y=r`）；经验回放 `10^5`、批量 `1000`、`MAX_GAMES=5000`；探索 `epsilon = max(0, 80 - n_games)` 且 `randint(0,200) < epsilon`。


| 项目         | DQN                      | DDQN                                  | Dueling DQN                                           |
| ---------- | ------------------------ | ------------------------------------- | ----------------------------------------------------- |
| 脚本         | `agent_DQN.py`           | `agent_DDQN.py`                       | `agent_DuelingDQN.py`                                 |
| 模型         | `model_DQN.py`           | `model_DDQN.py`                       | `model_DuelingDQN.py`                                 |
| 折扣 γ       | 0.9                      | 0.9                                   | 0.9                                                   |
| 目标网络       | 无；用当前网络对下一状态取 max Q      | 有在线网 + 目标网；每 **10** 局将在线权重复制到目标网      | 无；用当前网络对下一状态取 max Q                                   |
| TD 目标要点    | `y = r + γ·max Q_在线(s')` | 在线网选 `a* = argmax Q`，目标网估 `Q(s', a*)` | `Q = V(s) + A(s,a) - mean(A)`，bootstrap 用 `max Q(s')` |
| 损失         | MSE；仅覆写本步执行动作对应的 Q 维     | MSE；仅覆写本步执行动作对应的 Q 维                  | MSE；仅覆写本步执行动作对应的 Q 维                                  |
| 学习率        | 0.001                    | 0.001                                 | 0.001                                                 |
| 环境 shaping | 无（仅用环境 +10 / -10）        | 无                                     | 无                                                     |
| 相对优势       | 实现最简单                    | 减轻 Q 值过估计                             | 分离状态好坏与动作相对优劣                                         |


---

### B. 策略梯度 / Actor–Critic（PPO / TRPO / A2C）

**共性**：Actor/Critic 均为 `11→256`；**每局结束**更新（on-policy）；`γ=0.99`，`gae_lambda=0.95`；含距离 shaping（+0.5 / -0.2）；`MAX_GAMES=5000`。


| 项目        | PPO                                  | TRPO                                     | A2C                                   |
| --------- | ------------------------------------ | ---------------------------------------- | ------------------------------------- |
| 脚本        | `agent_PPO.py`                       | `agent_TRPO.py`                          | `agent_A2C.py`                        |
| 策略输出      | Actor 输出三动作 Softmax 概率               | Actor 输出三动作 Softmax 概率                   | Actor 输出三动作 Softmax 概率                |
| 核心机制      | 策略比率裁剪（`clip=0.2`）限制更新幅度             | 平均 KL ≤ 0.01；共轭梯度 + 线搜索更新 Actor          | 策略梯度 `log π(a                         |
| Actor 优化  | Adam，学习率 `3e-4`                      | 自然梯度方向 + 线搜索（非 Adam 小步）                  | Adam，学习率 `1e-3`                       |
| Critic 优化 | Adam，学习率 `1e-3`                      | Adam，学习率 `1e-3`，每局 **10** 个 epoch        | Adam，学习率 `1e-3`，每局 **10** 个 epoch     |
| 每局更新轮数    | `ppo_epochs=10`，mini-batch 大小 **64** | Critic **10** epoch；Actor 信任域内更新 **1** 次 | Actor **1** epoch；Critic **10** epoch |
| 探索        | 训练时 Categorical 采样；熵系数 **0.01**      | 训练时 Categorical 采样                       | 训练时 Categorical 采样                    |
| UI 推理     | 对 Softmax 概率取 `argmax`（贪心）           | 对 Softmax 概率取 `argmax`（贪心）               | 对 Softmax 概率取 `argmax`（贪心）            |
| 特点        | 工程上常用，稳定性较好                          | 单次更新保守，实现与计算更重                           | 结构轻，对超参较敏感                            |

### C. 图搜索（每步重规划：只走最短路的第一步）

**共性**：四邻接网格；墙 + 蛇身为障碍（各算法对蛇尾格处理见本表「蛇尾障碍」行）；无路时 `safe_random_action`（直/右/左，**忽略蛇尾** 试撞）。


| 项目   | BFS                | Dijkstra               | A*                  |
| ---- | ------------------ | ---------------------- | ------------------- |
| 脚本   | `play_bfs.py`      | `play_Dijkstra.py`     | `play_Ax.py`        |
| 数据结构 | 广度优先队列             | 按距离排序的小根堆              | 按 `f=g+h` 排序的小根堆    |
| 边权   | 每格移动代价为 1          | 每格移动代价为 1（代码可改为加权）     | 每格移动代价为 1           |
| 启发 h | 不使用                | 不使用                    | 蛇头到食物的曼哈顿距离         |
| 蛇尾障碍 | 墙 + 蛇身**全部格子**（含尾） | 墙 + 蛇身全部格子（含尾）         | 墙 + 蛇身；规划时**可剔除蛇尾** |
| 最优性  | 无权图下边数最少           | 边权均为 1 时，路径步数与广度优先搜索相同 | 启发可采纳时路径步数最少        |
| 失败兜底 | 直/右/左 试「忽略蛇尾」的安全动作 | 直/右/左 试「忽略蛇尾」的安全动作     | 直/右/左 试「忽略蛇尾」的安全动作  |


---

### D. 随机采样规划（RRT / RRT）


| 项目   | RRT                           | RRT*                                            |
| ---- | ----------------------------- | ----------------------------------------------- |
| 脚本   | `play_RRT.py`                 | `play_RRTx.py`                                  |
| 采样   | 随机网格坐标；**10%** 概率直接以食物为目标     | 随机网格坐标；**10%** 概率直接以食物为目标                       |
| 扩展   | 找树上距样本最近节点，向四邻中朝样本最近且未入树的一格生长 | 在 RRT 扩展基础上，对新点邻域 **重选父节点** 并 **rewire** 降低路径代价 |
| 迭代上限 | 每步最多 **1500** 次               | 每步最多 **1500** 次                                 |
| 蛇尾   | 墙 + 蛇身全部格子为障碍                 | 蛇身多于 1 节时，规划障碍中 **不占用蛇尾格**                      |
| 路径质量 | 能找到路即可，长度无保证                  | 迭代充分时路径趋向更短                                     |
| 适用   | 演示随机采样规划                      | 演示在随机树基础上重连、渐近缩短路径                              |

### E. 反应式（势场 / 局部打分）


| 项目   | APF                         | DWA（离散三动作版）                               |
| ---- | --------------------------- | ----------------------------------------- |
| 脚本   | `play_APF.py`               | `play_DWA.py`                             |
| 候选集  | 当前朝向前方、右转、左转三个相邻格           | 当前朝向前方、右转、左转三个相邻格                         |
| 目标   | 总势能最小                       | `α·heading + β·clearance + γ·velocity` 最大 |
| 默认参数 | `K_ATT=1`，`K_REP=10`，`D0=3` | `α=2`，`β=1`，`γ=0.5`                       |
| 障碍   | 墙 + **全身**蛇身                | 预判碰撞时 **忽略尾**                             |
| 优点   | 实现简单                        | 每步极快                                      |
| 缺点   | 局部极小、难贴目标                   | 无全局最优保证                                   |

### 场景选型（写报告 / 对战时参考）


| 需求           | 推荐算法            | 说明                                     |
| ------------ | --------------- | -------------------------------------- |
| 稳定吃豆、要最少步数倾向 | A*、Dijkstra、BFS | 全局最短路 + 滑窗重规划                          |
| 算力极省、可接受次优   | DWA、APF         | 每步 O(1)                                |
| 展示随机规划       | RRT、RRT*        | 注意 `max_iter` 有限可能失败                   |
| 从交互学习        | DQN 族、PPO 等     | 需训练；DQN 与 PPO 的 γ、shaping **不同**，宜分组对比 |
| 人机对战模型缺失     | 自动 BFS          | `RLPolicyController` 兜底                |


### 人机对战（`ui_ai_controllers`）


| 要点              | 说明                           |
| --------------- | ---------------------------- |
| `_ProxyGame`    | 对手占用格并入 `walls`，避免当成可释放的蛇尾   |
| `BFSController` | 模型加载失败时 RL 算法的回退策略           |
| 规划类             | 复用 `play_*.py` 中路径/势场/DWA 函数 |
